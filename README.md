☁️🚨 AWS CLOUD + SOC SECURITY LAB

📌 Overview

This project demonstrates a real-world Cloud Security Operations (SOC) environment using AWS services. It focuses on log monitoring, threat detection, and incident response using AWS-native security tools.

The lab simulates how modern SOC teams detect and respond to cloud-based threats.



🎯 Objectives

Implement cloud activity logging and monitoring
Detect suspicious behavior using AWS security tools
Analyze security incidents using CloudTrail logs
Investigate alerts generated by GuardDuty
Perform structured incident response
Document findings in SOC-style reports


🧰 AWS Services Used
AWS CloudTrail – API activity logging and auditing
Amazon GuardDuty – Threat detection and anomaly detection
Amazon CloudWatch – Monitoring and alerting
Amazon S3 – Secure log storage and retention
IAM (Identity and Access Management) – User and permission management


🏗️ Architecture Overview

The system follows a simple SOC cloud monitoring architecture:

AWS IAM Users
      ↓
AWS CloudTrail (Logs all API activity)
      ↓
Amazon S3 (Log Storage)
      ↓
Amazon GuardDuty (Threat Detection Engine)
      ↓
CloudWatch (Alerts & Monitoring Dashboard)
      ↓
SOC Analyst Investigation & Response
⚔️ Attack Simulation Scenarios

The following security scenarios were simulated to generate detection events:

Unauthorized login attempts from unfamiliar IP addresses
IAM user creation with elevated privileges
Suspicious API calls targeting S3 buckets
Cross-region access behavior anomalies

🔍 Detection & Monitoring
Key Detection Sources:
CloudTrail event logs (user activity tracking)
GuardDuty findings (threat intelligence alerts)
CloudWatch alarms (real-time monitoring triggers)
Example Findings:
Unusual login location detected
Suspicious IAM activity observed
Unauthorized API access attempts flagged

📊 Key Security Insights
Real-time detection of cloud threats using GuardDuty
Full visibility into AWS account activity via CloudTrail
Centralized monitoring through CloudWatch dashboards
Improved understanding of cloud attack patterns


🚨 Incident Response Workflow

Detection
GuardDuty identifies suspicious activity

Investigation
CloudTrail logs analyzed for root cause

Containment
IAM user access revoked or restricted

Eradication
Unauthorized credentials removed

Recovery
System restored to secure state
